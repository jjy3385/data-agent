# Feature Tasks 템플릿

> Tasks는 Approved Spec과 Plan을 기준으로 실제 수행한 구현·검증 작업과 리뷰 결과를 기록한다. 구현 전 작업 계획이나 승인 문서가 아니며, 새로운 요구사항이나 Plan에 없는 설계를 사후 정당화하지 않는다.

## Track별 작성 규칙

* `Lightweight`: 요구사항 Coverage, 실제 변경 파일과 동작, 실행한 검증, Plan 차이, 남은 위험과 Codex 검토 결과만 간결하게 기록한다. 같은 구현 결과와 완료 조건을 여러 절에 반복하지 않는다.
* `Standard`: 실제 작업을 검토 가능한 Task로 묶고 Task별 변경 파일, 정상·실패 검증, 실행 결과와 Plan 차이를 위험에 비례해 상세히 기록한다.

## 기록 원칙

* 실제로 완료하거나 시도한 작업만 기록하고 사전 계획처럼 작성하지 않는다.
* 코드 동작과 해당 테스트 또는 재현 가능한 검증을 같은 Task에 기록한다.
* 실행하지 않은 테스트를 통과로 표시하지 않는다.
* Approved Plan과 차이가 있으면 영향과 처리 결과를 명시한다.
* 중요한 설계, 공개 경계, 의존성 또는 파일 책임의 차이는 Tasks에서 승인하지 않고 Plan을 먼저 수정·재검토한다.
